What is Claude Code?

Claude Code runs on your machine, not in a browser. It can see your files, run commands, and make changes to your project. That makes it powerful — and it's why understanding safety and permissions matters.

Your safety net

You can always undo. Claude Code saves a snapshot of your code before every change. Press Esc twice or type /rewind to go back to any previous point. You can restore your code, your conversation, or both.

Essential commands

Type these at the Claude Code prompt.

/help
See all available commands, including any custom ones set up for your project.
When to use: You're not sure what's possible.
/clear
Start a completely fresh conversation. Your files and settings aren't affected — only the conversation resets.
When to use: Switching to a different task, or Claude seems confused.
/compact
Free up conversation space. Conversations have a length limit — think of it like a whiteboard that fills up. This summarizes older notes to make room.
When to use: Claude seems to forget things or is slowing down. (Also happens automatically.)
/cost
Check how much you've spent this session — token usage and dollar cost.
When to use: Keeping track of spending, especially on longer tasks.
/model
Switch which Claude model you're using. You can change models mid-conversation.
When to use: Switching to a faster model for quick questions, or a more thorough one for complex work.
Esc Esc
Open the rewind menu. Scroll through your prompts and restore code, conversation, or both to any previous point.
When to use: Something went wrong and you want to go back.

Setup & configuration

These help you configure how Claude Code behaves. Most are set-once.

/setup
Guided first-time setup. Walks you through permissions, model selection, and preferences with plain-language explanations.
When to use: First time using Claude Code, or want to reconfigure.
/config
Change settings interactively — model, output format, and other preferences.
When to use: You want to tweak a specific setting.
/allowed-tools
View and change what Claude can do without asking permission.
When to use: Claude asks too often (or not enough) before acting.
/init
Create a CLAUDE.md file — instructions that tell Claude how to behave in this project.
When to use: Setting up a new project, or you want to customize Claude's behavior.
/memory
Manage what Claude remembers across sessions — project knowledge that persists.
When to use: You want to see or edit what Claude has learned about your project.
/hooks
Configure automated checks that run when Claude takes certain actions.
When to use: You want to enforce rules (like requiring explanations before actions).

Advanced commands

/doctor
Diagnose problems with your Claude Code setup.
When to use: Something seems broken — commands fail or things aren't working.
/add-dir
Give Claude access to additional folders beyond the current project.
When to use: Your work spans multiple projects or repositories.
/agents
Create and manage helper agents that can work on tasks in parallel.
When to use: Complex tasks that benefit from multiple agents working simultaneously.
/mcp
Connect Claude to external tools and services via the Model Context Protocol.
When to use: You want Claude to interact with specific external tools.

Models

Switch anytime with /model. Use faster models for quick tasks to save money.

ModelBest forSpeedCost
Opus Complex tasks, multi-file changes, deep reasoning Slower Higher
Sonnet Everyday work — good balance of capability and speed Medium Medium
Haiku Quick questions, simple edits, fast searches Fast Lower

Tip: Claude can also use different models for subtasks automatically. When it spins up a helper agent for a quick search, it might use Haiku even if you're on Opus — saving you money without you having to think about it.

Permissions

Permissions control what Claude can do without asking you first.

Deny

Blocked entirely. Can't be overridden. Protects against dangerous mistakes.

Ask

Claude asks you each time. You see what it wants to do and can say yes or no.

Allow

Claude does it automatically. Speeds up routine work you trust.

Deny always wins — if something is blocked, no allow rule can override it. Run /setup to choose a permission profile, or see PERMISSIONS.md for the full reference.

Key concepts

Context window

How much Claude can hold in working memory. Like a desk — when it fills up, older notes get summarized. Use /compact to free space, or /clear to start fresh.

Skills

Custom commands (like /setup) that extend what Claude can do in a project. Defined in .claude/skills/.

Hooks

Automated checks that run when Claude takes actions. Can enforce rules, validate changes, or log what happened.

CLAUDE.md

A file of instructions Claude follows in this project. Like a briefing document — "here's how we do things here."

Persistent memory

Notes Claude keeps across sessions in MEMORY.md. This is how Claude learns your project over time. Manage with /memory.

Checkpoints

Automatic snapshots before every change. Press Esc twice to browse them and restore code, conversation, or both.

Common situations

"Claude is acting slow or forgetting things"
Your context window may be filling up. Try /compact to free space, or /clear to start fresh if your current task is done.
"I want to undo what Claude just did"
Press Esc twice to open the rewind menu. Pick the point you want to go back to. You can restore just the code, just the conversation, or both.
"Claude keeps asking for permission and it's slowing me down"
Run /setup and choose a more permissive profile (Standard or Open), or use /allowed-tools to grant specific permissions.
"I don't know if what Claude is doing is safe"
Look for the [permission_explanation] tag — Claude is required to explain every new action in plain language before doing it. If you're unsure, just say "stop" or "explain what that would do."
"I'm worried about costs"
Run /cost to check current spending. To reduce costs: use Haiku (/model) for simple questions, start fresh sessions (/clear) when switching tasks, and be specific in your requests.
"Something seems broken"
Run /doctor to diagnose setup issues. If that doesn't help, try /clear for a fresh session.
